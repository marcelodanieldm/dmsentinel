Protocolo DM Sentinel

Guia técnico sobre o processo de auditoria, mitigação e critérios de pontuação de risco.

Fluxo de Trabalho do Serviço

1

Ativação via Pagamento Global

O processo é iniciado instantaneamente após a confirmação do pagamento via Stripe, Mercado Pago ou Pix.

2

Sentinel Engine: Varredura Profunda

Nosso motor Python executa um fingerprinting não invasivo para identificar versões de CMS e arquivos expostos (.env, .git).

3

Cruzamento de Inteligência (CVE)

As versões detectadas são contrastadas com nosso banco de dados Common Vulnerabilities and Exposures (CVE).

4

Alertas e Relatórios

Se o Score de Risco for inferior a 70, um alerta vermelho é enviado via Telegram. Os dados alimentam o seu dashboard no Power BI.

5

Sentinel Academy Onboarding

Credenciais de acesso ao Moodle são enviadas para iniciar o treinamento obrigatório de higiene digital.

Algoritmo de Scoring

O Sentinel Score começa em 100 pontos. Deduções são feitas com base no impacto potencial à segurança.

Categoria Impacto (Dedução) Exemplo de Achado
Crítica-40 PontosVersão de CMS vulnerável a RCE.
Alta-20 PontosArquivos sensíveis expostos ou SSL expirando.
Média-10 PontosAusência de cabeçalhos de segurança (CSP, HSTS).
Baixa-5 PontosExposição da versão do servidor.
85-100Seguro
70-84Aviso
< 70Crítico
